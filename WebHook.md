- HTTP/REST
    - Long Polling
    - Timer Driven
- Web Socket
    - Open Connection 24/7
    - Event Driven
    - PUSH
- Web Hook
    - PUSH
    - Event Driven
    - No 24/7 Open Connection Required

How Web Hook Works
    - Client registers its Web Hook URL to Server API
    - When there is an event in server, Server invokes the client's web hook URL to Notify.
